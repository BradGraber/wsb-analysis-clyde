# Story: Seed Configuration Data with 34 System Config Entries

## Description
Insert all 34 system_config entries with their default values into the system_config table, organized across 7 configuration groups. The seed data script must match the exact key names and default values defined in PRD Section 3.3.4 and the initialization SQL in Appendix B. This enables the configuration-driven design where all thresholds and weights are read from the database at runtime.

## Acceptance Criteria
- [ ] All 34 system_config entries are inserted with correct key names and default values matching PRD Section 3.3.4: 2 system state entries (system_start_date, phase), 5 signal threshold entries (quality_min_users=2, quality_min_confidence=0.6, consensus_min_comments=30, consensus_min_users=8, consensus_min_alignment=0.7), 4 confidence weight entries (volume=0.25, alignment=0.25, ai_confidence=0.30, author_trust=0.20), 6 author trust entries (quality=0.40, accuracy=0.50, tenure=0.10, default_accuracy=0.50, tenure_saturation_days=30, accuracy_ema_weight=0.30), 10 stock exit entries, 6 options exit entries, 1 position management entry (signal_min_confidence=0.50)
- [ ] The 4 confidence weight values sum to exactly 1.0 (0.25 + 0.25 + 0.30 + 0.20)
- [ ] system_start_date is set to the current date (DATE('now')) at initialization time
- [ ] phase is set to 'paper_trading' as the default initial state
- [ ] Stock exit defaults match PRD: stop_loss=-0.10, take_profit=0.15, trailing_stop=0.07, breakeven_trigger=0.05, breakeven_min_days=5, time_stop_base_days=5, time_stop_base_min_gain=0.05, time_stop_extended_days=7, time_stop_max_days=10, take_profit_exit_pct=0.50
- [ ] Options exit defaults match PRD: stop_loss=-0.50, take_profit=1.00, trailing_stop=0.30, time_stop_days=10, expiration_protection_dte=2, take_profit_exit_pct=0.50
- [ ] Seed script is idempotent (re-running does not duplicate entries or error out)

## Technical Notes
- Exact INSERT SQL is provided in PRD Appendix B (lines 1842-1882)
- The seed script should use `INSERT OR IGNORE` or check for existing keys to support idempotent execution
- 33 keys from the PRD are enumerated in Section 3.3.4; the 34th key (signal_min_confidence=0.50) is added for qualified signal filtering (story-005-002, PIPE-032)
- The updated_at column should be set to DATETIME('now') for all entries at initialization
- This script runs after the schema creation (story-001-001) completes

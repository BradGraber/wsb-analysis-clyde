# Task: Add HITL override dropdown to prediction rows

## Description
Add a HITL override dropdown to each prediction row in the predictions table. Options: "No Override" (default), "Correct", "Incorrect", "Excluded". Selecting an option calls PATCH /predictions/{id}/override and updates the row display on success. Override is idempotent. Show inline error feedback if the PATCH fails.

## Acceptance Criteria
- [ ] Override dropdown on each prediction row with options: "No Override", "Correct", "Incorrect", "Excluded"
- [ ] Selecting an option calls PATCH /predictions/{id}/override with `{override_result: value}`
- [ ] Row display updates on success (is_correct column reflects new value)
- [ ] Override is idempotent: selecting same value twice produces no error
- [ ] Inline error feedback shown if PATCH fails (network error, validation error)
- [ ] "No Override" selection clears any existing override

## Technical Notes
- PATCH /predictions/{id}/override body: `{override_result: "correct"|"incorrect"|"excluded"}`
- "No Override" maps to removing the override (or PATCH with null)
- Use bootstrap-vue-next `<b-form-select>` for the dropdown
- On success: update local prediction data to reflect new is_correct value without full refetch
- On error: show inline error text below the dropdown, allow retry

## Dependencies
- Depends on: task-008-015-01 (prediction table provides the rows)
- Blocks: none

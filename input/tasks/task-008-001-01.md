# Task: Initialize Vue 3 project with Vite, Router, and Bootstrap integration

## Description
Scaffold a Vue 3 project using Vite with Composition API. Configure Vue Router with `createWebHistory` for 4 portfolio routes (Stock/Quality as default, Stock/Consensus, Options/Quality, Options/Consensus). Install and configure bootstrap-vue-next. Set up Vite proxy for `/api` to `http://localhost:8000`. Establish project directory structure: `src/views/`, `src/components/`, `src/composables/`, `src/api/`.

## Acceptance Criteria
- [ ] Vue 3 project scaffolded with Vite, Composition API only (no Options API)
- [ ] Vue Router configured with 4 portfolio routes; `/` redirects to Stock/Quality
- [ ] bootstrap-vue-next installed and applied to layout shell
- [ ] Vite proxy configured: `/api` -> `http://localhost:8000`
- [ ] Directory structure created: `src/views/`, `src/components/`, `src/composables/`, `src/api/`
- [ ] `npm run dev` starts local dev server and renders layout shell

## Technical Notes
- Use `createRouter` with `createWebHistory` for clean URLs
- Routes: `/stock-quality` (default), `/stock-consensus`, `/options-quality`, `/options-consensus`
- No localStorage, sessionStorage, Vuex, or Pinia anywhere
- Desktop-optimized: assume minimum viewport 1280px, no responsive breakpoints
- NFR-010: No authentication middleware
- Layout shell: header with app title + navigation area, `<router-view>` content area
